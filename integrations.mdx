## Overview

Glitch supports seamless integrations with popular third-party services, webhooks for real-time notifications, embedding capabilities, and a robust API for custom extensions. You can automate workflows, sync content across platforms, and extend functionality without leaving your documentation space.

Use these features to connect Glitch to tools like Slack for updates, GitHub for version control, or Zapier for no-code automations.

<Callout kind="info">
  Review your Glitch project settings before enabling integrations. Ensure you have admin access.
</Callout>

## Setting Up Webhooks

Webhooks enable real-time notifications when events occur, such as document updates or publishes. Configure them in your project settings.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Go to `https://dashboard.example.com/projects/{projectId}/webhooks` and click "New Webhook".
  </Step>
  <Step title="Add URL" icon="link">
    Enter your endpoint URL, like `https://your-webhook-url.com/glitch`.
  </Step>
  <Step title="Select Events" icon="settings">
    Choose events: `doc.updated`, `doc.published`, `comment.added`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and save.
  </Step>
</Steps>

Example webhook payload:

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  {
    "event": "doc.updated",
    "projectId": "proj_123abc",
    "docId": "doc_xyz789",
    "timestamp": "2024-01-15T10:30:00Z",
    "user": {
      "id": "user_456def",
      "email": "user@example.com"
    }
  }
  ```
  ```python
  payload = {
      "event": "doc.updated",
      "projectId": "proj_123abc",
      "docId": "doc_xyz789",
      "timestamp": "2024-01-15T10:30:00Z",
      "user": {
          "id": "user_456def",
          "email": "user@example.com"
      }
  }
  ```
</CodeGroup>

## Embedding External Content

Embed live content from external sources directly into your Glitch docs using iframes or API-driven components.

<Tabs>
  <Tab title="Iframe Embed" icon="layout">
    Use this for dashboards or tools:

    ```
    <Iframe
      src="https://dashboard.example.com/embed/{projectId}"
      title="Glitch Project Dashboard"
      width="100%"
      height="400"
    />
    ```

    Replace `{projectId}` with your actual ID.
  </Tab>
  <Tab title="API Embed" icon="code">
    Fetch data dynamically:

    <CodeGroup tabs="JavaScript,React">
      ```javascript
      async function embedContent(projectId) {
        const response = await fetch(`https://api.example.com/v1/projects/${projectId}/stats`);
        const data = await response.json();
        // Render data
      }
      ```
      ```jsx
      import { useEffect, useState } from 'react';

      function StatsEmbed({ projectId }) {
        const [stats, setStats] = useState(null);
        useEffect(() => {
          fetch(`https://api.example.com/v1/projects/${projectId}/stats`)
            .then(res => res.json())
            .then(setStats);
        }, [projectId]);
        return <div>{/* Render stats */}</div>;
      }
      ```
    </CodeGroup>
  </Tab>
</Tabs>

## API for Custom Extensions

Build custom integrations using Glitch's REST API at `https://api.example.com/v1`.

<ParamField path="projectId" param-type="string" required="true">
  Your Glitch project identifier.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `YOUR_API_KEY`.
</ParamField>

Example: List documents.

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X GET https://api.example.com/v1/projects/{projectId}/docs \
    -H "Authorization: Bearer YOUR_API_KEY"
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/projects/{projectId}/docs', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  });
  const docs = await response.json();
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "docs": [
      {
        "id": "doc_xyz789",
        "title": "Integrations Guide",
        "updatedAt": "2024-01-15T10:30:00Z"
      }
    ]
  }
  ```
</Response>

<Expandable title="Advanced API Usage" default-open="false">
  For webhooks via API:

  ```javascript
  await fetch(`https://api.example.com/v1/projects/{projectId}/webhooks`, {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      url: 'https://your-webhook-url.com/glitch',
      events: ['doc.updated']
    })
  });
  ```
</Expandable>
